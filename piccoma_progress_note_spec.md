# 続きどこ? (Where Was I?) 機能仕様書

最終更新: 2026-06-15
作成者: よう / Claude

## 1. 概要

### 1.1 目的
ピッコマをPCとモバイルで交互に利用する際、「どの作品を何話まで読んだか」「待てば0円のチャージ状況」が分からなくなる問題を解消する、個人用の進捗管理Webアプリ。アプリ名「続きどこ?」(英語名: Where Was I?)は、その瞬間の戸惑いをそのまま表現したもの。

### 1.2 背景・課題
- ピッコマWeb版は複数端末・複数ブラウザで同時ログインができず、片方の端末でログインすると他方が自動ログアウトされる仕様がある。
- そのため「PCで読んだ後にスマホで開く」といった使い方をすると、どこまで読んだか・チケットを使ったかが分かりにくくなる。
- 「待てば0円」は作品ごとに23時間のチャージ制で、複数作品を並行して読んでいると「今日どの作品が無料で読めるか」を把握しづらい。
- 既存の読書管理アプリ(コミックログ等)は単行本の巻数管理が主目的で、話数単位・チケット運用には対応していない。

### 1.3 ターゲットユーザー
- ピッコマで複数作品を並行購読しているユーザー(まずは本人=よう様)。
- PC/モバイルを切り替えながら利用する習慣がある。

### 1.4 スコープ(MVP)
本仕様書はMVP(Minimum Viable Product)を対象とする。自動取得・他アプリ対応・ソーシャル機能はPhase 2以降とする。

## 3. データモデル

### 3.1 works (作品)

| フィールド | 型 | 説明 |
|---|---|---|
| id | string | 作品ID(自動生成) |
| title | string | 作品名 |
| genre_tags | string[] | ジャンルタグ(任意) |
| piccoma_url | string | ピッコマ作品ページURL |
| cover_image_url | string | サムネイル画像URL(任意、手動入力) |
| created_at | timestamp | 登録日時 |
| archived | boolean | 完読・追跡終了フラグ |

### 3.2 progress (進捗)

| フィールド | 型 | 説明 |
|---|---|---|
| work_id | string | 紐づく作品ID |
| current_episode | number | 現在の既読話数 |
| next_episode | number | 次に読む話数(基本は current_episode + 1) |
| updated_at | timestamp | 最終更新日時 |
| updated_device | string | 更新元デバイス("PC" / "Mobile") |
| memo | string | 任意メモ(例: 「ここから話が動く」など) |

### 3.3 tickets (待てば0円チケット)

| フィールド | 型 | 説明 |
|---|---|---|
| work_id | string | 紐づく作品ID |
| used_at | timestamp | チケットを使用した日時 |
| charge_complete_at | timestamp | チャージ完了予定(used_at + 23h、自動計算) |
| status | string | "charging" / "ready" / "used_today"(計算で導出可) |

### 3.4 progress_history (更新履歴・任意)

| フィールド | 型 | 説明 |
|---|---|---|
| work_id | string | 紐づく作品ID |
| episode | number | 更新後の話数 |
| changed_at | timestamp | 変更日時 |
| device | string | 更新元デバイス |

## 4. 機能要件

### 4.1 作品管理
- 作品の新規登録(作品名・ジャンルタグ・ピッコマURL・サムネイル画像URL)
- 作品一覧表示(検索・並び替え: 最終更新日順／チャージ完了が近い順)
- 作品の編集・アーカイブ(完読時など)

### 4.2 進捗記録
- 作品ごとに現在の既読話数を表示
- 「+1話読んだ」ワンタップ更新ボタン
- 更新時に updated_at / updated_device を自動記録
- 話数の手動補正(直接数値入力での上書き)
- 任意メモの追加

### 4.3 待てば0円チケット管理
- 「待てば0円を使った」ボタンをタップすると used_at を記録し、charge_complete_at(used_at+23h)を自動計算
- チャージ完了までの残り時間をカウントダウン表示
- チャージ完了済みの作品は「読める」ステータスで強調表示
- (PWA対応時)チャージ完了時にWeb Push通知でリマインド

### 4.4 ダッシュボード(ホーム画面)
最初に表示される画面。以下を一覧表示する。
- 「今日待てば0円が使える作品」セクション(チャージ完了済みを優先表示)
- 「チャージ中の作品」セクション(残り時間を表示、近い順にソート)
- 各作品の現在の既読話数・次に読むべき話数・最終更新情報(日時+デバイス)

### 4.5 デバイス間の状態共有
- Firebase Authenticationでログインし、PC/モバイルどちらからアクセスしても同一データを参照
- ピッコマ本体のログイン排他とは独立して、進捗・チケット情報は常に同期

### 4.6 履歴・補正
- 進捗の編集機能(誤入力の修正)
- 簡易な更新履歴の表示(いつ・どの話数に・どのデバイスから更新したか)

## 5. UI仕様

### 5.1 デザイン方針
- 「見やすくて単純」を最優先。情報量を絞り、一目で「今どうすればいいか」が分かる構成にする。
- カラーは状態を直感的に伝える配色とする(例: 緑系=今すぐ読める / グレー系=チャージ中)。
- スマホでの片手操作を想定し、主要アクション(+1話、チケット使用)はタップしやすい大きさ・位置に配置。

### 5.2 画面構成

#### ダッシュボード(ホーム)
- 上部: 「今すぐ読める作品」カード一覧(横スクロールまたは縦並び)
  - 作品サムネイル、作品名、現在話数、「+1話読んだ」ボタン、「待てば0円を使った」ボタン
- 中部: 「チャージ中」セクション
  - 作品名、残り時間(カウントダウン表示)
- 下部: 全作品への一覧リンク

#### 作品一覧画面
- 検索バー
- 並び替えセレクタ(更新日順／チャージ完了順)
- 作品カードのリスト(進捗・チャージ状況を一目で確認できる簡易表示)

#### 作品詳細画面
- 作品情報(名前・タグ・ピッコマURLへのリンク)
- 進捗編集(現在話数の手動入力)
- チケット使用履歴・チャージ完了予定
- 更新履歴(簡易タイムライン)

#### 作品登録/編集画面
- フォーム形式(作品名・タグ・URL・サムネイルURL)

### 5.3 UI実装について
具体的なワイヤーフレーム・コンポーネント設計はClaude側で作成する。Next.js + Tailwindでのレスポンシブ実装を前提とし、まずダッシュボード画面のモックから着手する想定。

---

## 6. 非機能要件

- 個人利用前提のため、複数ユーザー対応は不要(将来的な拡張余地として認証は残す)
- データ量は小規模(作品数十件程度)を想定し、Firestoreの無料枠内で運用可能
- オフライン時の表示は考慮しない(MVP時点)
